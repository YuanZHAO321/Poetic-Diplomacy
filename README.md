# Poetic Diplomacy · 赋诗言志

> 子曰："不学《诗》，无以言。"

一个 [Claude Code Skill](https://docs.claude.com/en/docs/claude-code/skills)，复原春秋时代"赋诗言志"的古外交艺术——列国卿大夫在聘问、盟会、享宴上不直陈其事，而是诵《诗经》某篇某章，借断章之义完成求援、修好、施压、拒绝与观志。一首诗赋得恰当可以借来军队（秦哀公赋《无衣》而出师救楚），赋得不当则当场结怨甚至亡身（伯有赋《鹑之贲贲》，赵孟预言"将为戮矣"）。

本技能**不是凭模型记忆写古风**：它由《左传》《诗经》《尚书》三部原典全文驱动，构建时从《左传》全书系统提取了 **83 处赋 / 歌 / 诵《诗》记录，合并为 30 个独立事件全部收录**，并将《诗经》全文打包为可检索资产——技能运行时引诗必先查原文，逐字核对，杜绝错引。

## 它能做什么

| 模式 | 说明 | 试试这样问 |
|---|---|---|
| **拟赋** | 为一个外交意图挑选诗篇、设计断章与应答链 | "我是小国使者要去大国求援，帮我设计享礼上的赋诗" |
| **解赋** | 解读他人赋诗的弦外之音，给出回应方案 | "楚国令尹在宴会上赋了《大明》首章，什么意思？多严重？" |
| **演礼** | 生成一整场聘问 / 盟会 / 享宴 / 饯行的完整剧本 | "模拟郑国六卿为晋国执政饯行的赋诗全场" |
| **评赋** | 像赵孟、叔向那样判"类不类、僭不僭"，据诗观志、预言吉凶 | "有人在两君相见时赋了《湛露》，评一评" |
| **辞命** | 撰写散文体行人辞令与《尚书》体誓、诰、命 | "帮我写一篇吕相绝秦式的绝交书" |

也支持现代场景转译——把商务谈判、催款邮件、团队动员写成春秋外交体：合作 = 修好，催款 = 责贡，挖角 = 《褰裳》之讽。

## 安装

将 `poetic-diplomacy/` 目录复制到 Claude Code 的技能目录即可：

```bash
# 个人全局安装
cp -r poetic-diplomacy ~/.claude/skills/

# 或仅在某个项目内使用
cp -r poetic-diplomacy <your-project>/.claude/skills/
```

之后在 Claude Code 中提及"赋诗""春秋外交""行人辞令"等话题时技能会自动触发。

## 技能结构

```
poetic-diplomacy/
├── SKILL.md                    # 入口：五种任务模式、工作流程、核心规则速览、红线
├── references/
│   ├── protocol.md             # 礼法规则书：仪程、断章取义、歌诗必类、等级限度、
│   │                           #   答赋义务与沉默的语义、赋诗即承诺、观志术、称谓套语
│   ├── episodes.md             # 《左传》赋诗事例全库：30 事件编年全收，
│   │                           #   十大名场面详注（垂陇观志、六卿饯宣子、申包胥哭秦庭…）
│   ├── repertoire.md           # 外交用诗索引：按意图分类（乞师/修好/情诗转外交/施压/讽刺），
│   │                           #   附《左传》篇名↔今本异文对照与逸诗表
│   └── rhetoric.md             # 散文辞令：《尚书》誓诰命文体谱系 + 《左传》七大辞令范式
├── scripts/
│   └── find_poem.py            # 《诗经》查询工具：按篇名或诗句检索，自动提示同名异篇
├── assets/
│   └── shijing.txt             # 《诗经》全文（305 篇，技能运行时的引诗校对源）
└── evals/
    └── evals.json              # 测试用例（可选，供 skill-creator 迭代评估）
```

```bash
# 查诗工具单独可用
python3 poetic-diplomacy/scripts/find_poem.py 褰裳          # 按篇名查全文
python3 poetic-diplomacy/scripts/find_poem.py --grep 同袍   # 按诗句反查篇目
```

## 构建方法：从原典到技能

1. **通读语料**：仓库根目录的 `左传.txt`（767 KB）、`诗经.txt`（324 KB）、`尚书.txt`（99 KB）为三部原典全文。
2. **穷举提取**：以正则扫描《左传》全文中所有 `赋/歌/诵《…》` 记录（83 处），按位置聚类合并为 30 个独立事件，并补捉不带书名号的关键规则论述（"歌诗必类""赋诗断章，余取所求焉"等）。
3. **原文核实**：每条规则、每个事例、每句引诗均与原典逐字比对后才写入参考文档；发现的《左传》篇名与今本《诗经》异文（《嘉乐》=《假乐》、《鹑之贲贲》=《鹑之奔奔》等）整理为对照表。
4. **反幻觉设计**：《诗经》全文作为资产随技能分发，SKILL.md 强制要求"凡引诗先运行 `find_poem.py` 核对"；逸诗（《河水》《茅鸱》《新宫》《辔之柔矣》）明确标注，禁止伪造诗句。
