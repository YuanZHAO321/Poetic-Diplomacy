# Poetic Diplomacy · 赋诗言志

> 子曰："不学《诗》，无以言。"

一个 [Claude Code Skill](https://docs.claude.com/en/docs/claude-code/skills)，复原春秋时代"赋诗言志"的古外交艺术——列国卿大夫在聘问、盟会、享宴上不直陈其事，而是诵《诗经》某篇某章，借断章之义完成求援、修好、施压、拒绝与观志。一首诗赋得恰当可以借来军队（秦哀公赋《无衣》而出师救楚），赋得不当则当场结怨甚至亡身（伯有赋《鹑之贲贲》，赵孟预言"将为戮矣"）。

本技能**不是凭模型记忆写古风**：它由《左传》《诗经》《尚书》三部原典全文驱动。三部书各司其职——**《左传》是 log**（外交实战的全部记录），**《诗经》是核心表达语料库**（宴享之上直接赋诵），**《尚书》是政治论证语料库**（不直接赋诵，但如法典宪章般被"《书》曰"援引，为政治原则提供依据与正当性）。构建时从《左传》全书系统提取了 **83 处赋 / 歌 / 诵《诗》记录（合并为 30 个独立事件全部收录）、43 处引《书》、126 处"《诗》曰"引诗**，并将《诗经》《尚书》全文打包为可检索资产——技能运行时引经必先查原文，逐字核对，杜绝错引。

## 它能做什么

| 模式 | 说明 | 试试这样问 |
|---|---|---|
| **拟赋** | 为一个外交意图挑选诗篇、设计断章与应答链 | "我是小国使者要去大国求援，帮我设计享礼上的赋诗" |
| **解赋** | 解读他人赋诗的弦外之音，给出回应方案 | "楚国令尹在宴会上赋了《大明》首章，什么意思？多严重？" |
| **演礼** | 生成一整场聘问 / 盟会 / 享宴 / 饯行的完整剧本 | "模拟郑国六卿为晋国执政饯行的赋诗全场" |
| **评赋** | 像赵孟、叔向那样判"类不类、僭不僭"，据诗观志、预言吉凶 | "有人在两君相见时赋了《湛露》，评一评" |
| **辞命** | 撰写散文体行人辞令与《尚书》体誓、诰、命 | "帮我写一篇吕相绝秦式的绝交书" |

也支持现代场景转译——把商务谈判、催款邮件、团队动员写成春秋外交体：合作 = 修好，催款 = 责贡，挖角 = 《褰裳》之讽。

## 安装

将 `poetic-diplomacy/` 目录复制到 Claude Code 的技能目录即可：

```bash
# 个人全局安装
cp -r poetic-diplomacy ~/.claude/skills/

# 或仅在某个项目内使用
cp -r poetic-diplomacy <your-project>/.claude/skills/
```

之后在 Claude Code 中提及"赋诗""春秋外交""行人辞令"等话题时技能会自动触发。

## 构建方法：从原典到技能

1. **通读语料**：仓库根目录的 `左传.txt`（767 KB）、`诗经.txt`（324 KB）、`尚书.txt`（99 KB）为三部原典全文。
2. **穷举提取（表达层）**：以正则扫描《左传》全文中所有 `赋/歌/诵《…》` 记录（83 处），按位置聚类合并为 30 个独立事件，并补捉不带书名号的关键规则论述（"歌诗必类""赋诗断章，余取所求焉"等）。
3. **穷举提取（论证层）**：同法扫描《左传》全部引经记录——43 处引《书》（《书》曰/《夏书》曰/《周书》曰/《大誓》曰…）、126 处"《诗》曰/云"——按政治原则归类为论证引用库，还原《尚书》作为"法典宪章类文献"在外交论证中的真实角色。
4. **原文核实**：每条规则、每个事例、每句引文均与原典逐字比对后才写入参考文档。《左传》篇名与今本《诗经》异文（《嘉乐》=《假乐》等）、引《书》与今本《尚书》异文（如↔若、弗↔不、人↔民）均整理为对照规则；引文落在东晋伪古文篇者，明确标注"伪古文实为反向抄撮《左传》"，不倒果为因。
5. **反幻觉设计**：三部原典全文均作为资产随技能分发（技能自包含，脱离本仓库亦可校对），SKILL.md 强制要求"凡引经先运行 `find_poem.py` / `find_shu.py` 核对，援引《左传》原文先经 `find_zuo.py` 核实"；逸诗（《河水》《茅鸱》《新宫》《辔之柔矣》）与逸书（"居安思危""父子兄弟罪不相及"）明确标注，禁止伪造扩写。参考文档是蒸馏层，三个查询接口是原文层——事例库覆盖不到的细节一律回原文取证。
